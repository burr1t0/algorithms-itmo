## 2. Реализованные алгоритмы

### 2.1 Пузырьковая сортировка (Bubble Sort)
- **Сложность:** O(n²)
- **Принцип работы:** Многократный проход по массиву с попарным сравнением и обменом соседних элементов
- **Особенности:** Простая реализация, но очень медленная на больших данных

### 2.2 Быстрая сортировка (Quick Sort)
- **Сложность:** O(n log n) в среднем случае
- **Принцип работы:** Рекурсивное разделение массива относительно опорного элемента
- **Особенности:** Эффективный алгоритм для общего применения

### 2.3 Сортировка подсчётом (Counting Sort)
- **Сложность:** O(n + k), где k — диапазон значений
- **Принцип работы:** Подсчёт количества каждого элемента и восстановление отсортированного массива
- **Особенности:** Линейная сложность, но требует ограниченного диапазона значений

## 7. Практические наблюдения

### 7.1 Ограничения Counting Sort
Хотя Counting Sort показал лучшую производительность, у него есть важные ограничения:
- Работает только с целыми числами
- Требует знания максимального значения
- Потребляет дополнительную память O(k), где k — диапазон значений

### 7.2 Оптимальный выбор алгоритма
- **Малые данные (N < 1000):** разница между алгоритмами несущественна
- **Средние данные (1000 < N < 10000):** Quick Sort оптимален
- **Большие данные (N > 10000):** только Quick Sort или специализированные алгоритмы
- **Ограниченный диапазон значений:** Counting Sort — лучший выбор

## 8. Выводы

2. **Квадратичные алгоритмы (O(n²)) непригодны** для обработки больших объёмов данных — время выполнения становится непрактичным уже при N > 10000.

4. **Линейные алгоритмы (O(n)) показывают максимальную скорость**, но имеют серьёзные ограничения на применение.

5. **Разница в производительности огромна** — выбор алгоритма может ускорить выполнение задачи в тысячи раз.
